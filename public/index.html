Add Robots
We start with the provocation: why can't workers own their robots?

The Idea

When you work backward from the idea that workers should own rather than fear robots, you end up making different choices.


We started down this path in 2016 after repeated disappointment that the tech industry was hell-bent to replace all workers without even the slightest moral concern.

The idea is built up from a new kind of robot control motor that is a USB 3.1 Type-C Android peripheral.

Plug them into Android and they get data, 100W power and cloud control over the USB cable.

And because they're Android-based, they're inherently mobile, cheap, secure and sensor-rich.

The robot behaviors live in the cloud, are communal and shared through remixing.

We don't have all the answers, but after a bunch of work, we have a good start:

Stupid simple

Inherently cloud-controlled

Inherently communal with effortless remixing just like Scratch

Primarily Javascript (with options for any language)

100% open: software and hardware

Mix-and-match parts that are cheap, and user-reparable

Based on "previous generation smartphones" to keep the cost down and deal with the environmental problem

Durable and rugged

Owned by the community

Motor Control Unit

The motor control unit is a Kinetis K22F 120MHZ ARM Cortex M4 that communicates motor and sensor control commands over USB-CDC using Google Protocol Buffers. The interface clocks at 1.25MBps so that it is possible to get quick closed-loop PID control from collected sensor data.

Vehicle Control Unit

The vehicle control unit is an Android application that speaks to the web via SSH by command line or Google Protocol Buffers. It also provides a webcam, microphone/speaker remote-pilot option.

Power System

The power system is a fast-charging, 24V, 15Wh LiFePO4 battery with a 5V, 2A external power bus in USB form factor. This allows you to attach the Android and various sensors to the frame and draw power from the single source.
